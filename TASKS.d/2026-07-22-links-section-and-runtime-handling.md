# Task
Update `index.md` rendering in `bin/classroom-archiver.py` to include a `## Links` section for external web links and YouTube videos, and update material mapping/error handling for runtime execution.

## Scope
* Modified files: `bin/classroom-archiver.py`
* Modified tests: `tests/test_classroom_archiver.py`

## Requirements & Constraints

### 1. `index.md` Links Section
When rendering `index.md` for an assignment:
* If an assignment contains materials of type `link` or `youtubeVideo`, append a `## Links` section to `index.md` after the description.
* Format each link item as a Markdown bullet:
  * Links: `* [{title}]({url})`
  * YouTube Videos: `* [{title}]({url}) (YouTube)`
* If no external links or YouTube videos are attached to the assignment, omit the `## Links` heading entirely.

### 2. Live API Error Boundaries & HTTP Statuses
When communicating with Google API endpoints (Classroom / Drive / Forms):
* **HTTP 404 / Missing Files:** If a referenced Google Drive file or attachment ID returns an HTTP 404 / Not Found, raise:
  `RuntimeError: "Drive file or attachment not found: [{file_id}] in assignment '{assignment_title}'. Halting execution."`
* **HTTP 403 / Authorization Errors:** If a Drive file fails due to insufficient permissions or invalid OAuth scope, raise:
  `RuntimeError: "Permission denied for Drive file [{file_id}] in assignment '{assignment_title}'. Check OAuth scopes. Halting execution."`

## Acceptance Criteria
* `index.md` generated by `MarkdownCorpusWriter` includes `## Links` if external links or YouTube videos exist, and omits the section if none exist.
* `classroom-archiver.py` raises descriptive `RuntimeError` messages when encountering HTTP 404 or HTTP 403 API responses during Drive metadata fetches.
* `python3 -m pytest` passes completely with new unit tests for `## Links` formatting and API error trapping.
